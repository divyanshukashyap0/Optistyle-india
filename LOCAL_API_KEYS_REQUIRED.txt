
================================================================================
LOCAL API KEYS REQUIRED
================================================================================

This document lists the specific API keys and configuration values required to run the OptiStyle project on your local machine.

---

1. BASIC INFORMATION
--------------------------------------------------------------------------------
- All keys must be saved in a file named `.env` in the project root.
- Do NOT commit the `.env` file to Git.
- These keys are for LOCAL development only (localhost).

---

2. FRONTEND API KEYS (Required for Browser)
--------------------------------------------------------------------------------
These variables must start with "VITE_" to be visible to the React application.

[ REQUIRED ]
Key: VITE_API_URL
Used By: React App (to talk to Backend)
Value: http://localhost:5000/api

[ REQUIRED ]
Key: VITE_RAZORPAY_KEY_ID
Used By: Payment Page (Checkout)
Value: Your Razorpay *Test* Key ID (starts with rzp_test_)

[ REQUIRED - FIREBASE CONFIG ]
Used By: Login/Signup Screens
Keys:
- VITE_FIREBASE_API_KEY
- VITE_FIREBASE_AUTH_DOMAIN
- VITE_FIREBASE_PROJECT_ID
- VITE_FIREBASE_STORAGE_BUCKET
- VITE_FIREBASE_MESSAGING_SENDER_ID
- VITE_FIREBASE_APP_ID
Value: Copy these directly from Firebase Console > Project Settings.

---

3. BACKEND API KEYS (Required for Server)
--------------------------------------------------------------------------------
These variables are kept secret on the server side.

[ REQUIRED ]
Key: PORT
Used By: Express Server
Value: 5000 (Default)

[ REQUIRED - FIREBASE ADMIN ]
Used By: Database Access (Orders, Products)
Keys:
- FIREBASE_PROJECT_ID
- FIREBASE_CLIENT_EMAIL
- FIREBASE_PRIVATE_KEY
Value: From Firebase Console > Service Accounts.
Note: The Private Key must be wrapped in quotes "..." and handle newlines (\n).

[ REQUIRED ]
Key: RAZORPAY_KEY_ID
Key: RAZORPAY_KEY_SECRET
Used By: Payment Verification API
Value: Must match the account used in Frontend. Use *Test* Mode keys.

[ OPTIONAL - EMAIL ]
Used By: Sending Order Confirmations
Keys:
- GMAIL_CLIENT_ID
- GMAIL_CLIENT_SECRET
- GMAIL_REFRESH_TOKEN
- GMAIL_SENDER
Value: Gmail OAuth2 Credentials. If missing, emails will not send, but app will run.

[ OPTIONAL - AI & MAPS ]
Keys:
- AI_API_KEY (DeepSeek/OpenAI)
- GOOGLE_MAPS_API_KEY
Value: If missing, AI chat uses fallbacks and Maps uses mock data.

---

4. MINIMUM SET TO START PROJECT
--------------------------------------------------------------------------------
If you just want the app to open without crashing, you MUST have:

FRONTEND:
1. VITE_API_URL
2. All VITE_FIREBASE_* keys

BACKEND:
1. FIREBASE_PROJECT_ID
2. FIREBASE_CLIENT_EMAIL
3. FIREBASE_PRIVATE_KEY

*Consequences of missing keys:*
- Missing Razorpay: Checkout button will do nothing.
- Missing Email Keys: Server logs an error on checkout, but order is saved.
- Missing Firebase: You cannot log in or see products.

---

5. COMMON MISTAKES TO AVOID
--------------------------------------------------------------------------------
- Missing the "VITE_" prefix for frontend keys.
- Using "Live/Production" payment keys locally (Real money will be deducted).
- Pasting the Firebase Private Key without quotes (causes parsing errors).
- Changing the `.env` file and forgetting to restart the terminal.
